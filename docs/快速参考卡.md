# 团队记忆系统 · 快速参考卡

> 一页纸，贴群即用。完整步骤见 [管理员操作清单](管理员操作清单.md) / [组员操作清单](组员操作清单.md)。
> `<占位符>` 按实际情况替换。Mac/Linux 用 `python3`，Windows 用 `python`。

---

## 🔧 管理员（一次性，~10 分钟）

**占位符**：`<GITEA>` `<ORG>` `<ADMIN>` `<ADMIN_TOKEN>` `<TEAM_NAME>`

```bash
# 1. 建组织 + 两个仓库(空、私有)
curl -X POST <GITEA>/api/v1/orgs -H "Authorization: token <ADMIN_TOKEN>" -H "Content-Type: application/json" -d '{"username":"<ORG>"}'
curl -X POST <GITEA>/api/v1/orgs/<ORG>/repos -H "Authorization: token <ADMIN_TOKEN>" -H "Content-Type: application/json" -d '{"name":"team-memory","private":true,"auto_init":false}'
curl -X POST <GITEA>/api/v1/orgs/<ORG>/repos -H "Authorization: token <ADMIN_TOKEN>" -H "Content-Type: application/json" -d '{"name":"agent-team-memory","private":true}'

# 2. 推工具仓库
cd agent-team-memory && git remote add origin <GITEA>/<ORG>/agent-team-memory.git && git push -u origin main

# 3. 初始化记忆仓库 + 推首条
pip install -e .
team-memory init -p ~/team-memory -n "<TEAM_NAME>" -a <ADMIN>
#   编辑 ~/team-memory/.env → GITEA_REMOTE_URL=<GITEA>/<ORG>/team-memory.git
team-memory capture "团队记忆系统启动" -t project --via claude -a <ADMIN> -p ~/team-memory --note "<TEAM_NAME> 启动"
cd ~/team-memory && git add -A && git commit -m "init" && team-memory sync

# 4. 把组员加入 <ORG> 组织；让每人生成 token(勾 repository)；分配唯一英文短名
# 5. 把下面"组员"段发到团队群
```

## 👤 组员（每人，~3 分钟）

**占位符**：`<GITEA>` `<ORG>` `<短名>` `<TOKEN>`

```bash
# 1. clone 工具 + 一键接入
git clone <GITEA>/<ORG>/agent-team-memory.git && cd agent-team-memory
python3 scripts/onboard.py <GITEA>/<ORG>/team-memory.git <短名>     # Windows 换 python

# 2. 配 token(二选一)
git config --global credential.helper store       # 推荐: 首次 sync 输 Gitea用户名 + <TOKEN>
# echo "GITEA_TOKEN=<TOKEN>" >> ~/team-memory/.env   # 或写 .env

# 3. 验证
team-memory doctor -p ~/team-memory     # 期望: ✓ 仓库健康
team-memory web -p ~/team-memory        # 浏览器打开, 能看到团队记忆
```

## 📝 日常速查

```bash
# 沉淀(正式, 已确认):
team-memory capture "标题" -t decision --via claude -a <短名> --note "..." -p ~/team-memory
# 候选(AI/未验证, 先进 inbox 待审 → approve 转正式):
team-memory propose "标题" -t decision --via codex -a <短名> --confidence high --evidence "PR-123" -p ~/team-memory
team-memory review -p ~/team-memory && team-memory approve <id> -p ~/team-memory
#   类型: decision | error | skill | project | code   via: claude | codex | hermes | trae

# 读记忆
team-memory list -p ~/team-memory                          # 全部(来源/时间/版本)
team-memory show mem-20260625-<短名>-001 -p ~/team-memory   # 详情+版本历史
team-memory web -p ~/team-memory                           # 可视化+检索框

# 同步(沉淀后必做!)
cd ~/team-memory && git add -A && git commit -m "memory: 标题" && team-memory sync
```

**【可选】MCP（AI 对话中动态查/写记忆）**：

## ⚠️ 出错速查

| 现象 | 修复 |
|---|---|
| `author 必须含 ascii` | 短名用英文，`--author <短名>` 或 `.env` 的 `DEFAULT_AUTHOR` |
| `team-memory 不是命令`（Windows） | `python -m team_memory <命令>` |
| `sync` 说"未提交变更" | 先 `git add -A && git commit`，或 `team-memory sync --force` |
| `sync` 报冲突 | 按提示删冲突标记 → `git add <文件> && git rebase --continue`；或 `git rebase --abort` |
| `web` 打不开 | `team-memory web --port 8001` |
| push 403 | 未加入组织 / token 无写权限 |

**铁律**：capture 后必须 commit + sync ｜ secrets 不入库 ｜ 核心记忆(project/decisions)改走 PR
