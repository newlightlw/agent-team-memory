# 组员操作清单 — 接入团队记忆系统

> **给 AI 工具（Cursor/Claude Code/Codex/Trae/Hermes 等）执行**：你是团队成员。按本清单接入团队的共享记忆仓库并开始用。遇到 `<占位符>` 时向用户要真实值。整个流程约 5 分钟。

---

## 前置（向管理员要这几样）

| 项 | 说明 |
|---|---|
| `<GITEA>` | 团队 Gitea 地址，如 `http://gitea.example.com` |
| `<ORG>` | 组织名，如 `ai-team` |
| `<短名>` | **你的英文短名**（管理员分配，用于记忆 id，如 `zhang3`；不能中文） |
| `<TOKEN>` | 你的 Gitea access token（头像→设置→应用→生成，勾选 `repository`） |
| `<TOOL_REPO>` | 工具仓库地址 `<GITEA>/<ORG>/agent-team-memory.git` |

**本机要求**：Python ≥ 3.11；已装常用 AI 工具（Claude Code / Codex / Trae / Hermes 任一）；已装 git。

---

## 步骤

### ① clone 工具仓库 + 一键接入

**Mac / Linux**：
```bash
git clone <TOOL_REPO>
cd agent-team-memory
python3 scripts/onboard.py <GITEA>/<ORG>/team-memory.git <短名>
```

**Windows**（用 `python`，不是 `python3`）：
```powershell
git clone <TOOL_REPO>
cd agent-team-memory
python scripts/onboard.py <GITEA>/<ORG>/team-memory.git <短名>
```

脚本会自动：clone 记忆仓库到 `~/team-memory` → 装 team-memory 工具 → 配 `.env` → 桥接到 4 个 AI 工具 → 自检。

**验证**：末尾出现 `✓ 接入完成! 记忆仓库: ~/team-memory`。

### ② 配 token（推送到 Gitea 用）

```bash
# 方式 A(推荐, 一次配好永久免输):
git config --global credential.helper store
# 首次 sync 时输入: 用户名=你的Gitea用户名, 密码=<TOKEN>

# 方式 B(写 .env):
echo "GITEA_TOKEN=<TOKEN>" >> ~/team-memory/.env
```

### ③ 验证接入成功

```bash
team-memory doctor -p ~/team-memory     # 期望: ✓ 仓库健康
team-memory web -p ~/team-memory        # 浏览器自动打开, 能看到团队记忆 + 检索框
```

> Windows 若提示 `team-memory 不是命令`，改用 `python -m team_memory doctor -p ~/team-memory`（不依赖 PATH，最稳）。

## 日常使用

### 沉淀一条记忆（两种方式）

**方式 A — 命令行**：
```bash
team-memory capture "记忆标题" -t decision --via <当前工具> -a <短名> \
  --note "背景/结论/要点" -p ~/team-memory
# 类型选: decision(决策) / error(错误) / skill(流程) / project(项目) / code(代码)
# --via 填当前用的工具: claude / codex / hermes / trae
```

**方式 B — 对话中直接说"提交下"**（最省事）：
> 在任何 AI 工具里聊完一个有价值的话题，说一句**"提交下"** → auto-commit-memory skill 会自动总结（背景/结论/要点/案例/使用场景）→ 给你看草稿 → 你回"可以"才提交。

### 读取团队记忆

```bash
team-memory list -p ~/team-memory            # 所有记忆(来源/时间/版本)
team-memory show mem-20260625-zhang3-001 -p ~/team-memory   # 某条详情+版本历史
team-memory web -p ~/team-memory             # 浏览器可视化+检索
```

### 同步到团队（推自己的 + 拉队友的）

```bash
cd ~/team-memory
git add -A && git commit -m "memory(<类型>): <标题>"
team-memory sync -p ~/team-memory            # pull --rebase + push
```

> **重要**：`capture` 不会自动 commit。沉淀后必须 `git add/commit` 再 `sync`，否则推不上去。

---

## 工作约定

1. **capture 后 commit + sync**：沉淀的闭环
2. **核心记忆走 PR**：改 `project/` / `decisions/` 类记忆，提 PR 让人 review（有 PR 模板）
3. **冲突按提示走**：`sync` 报冲突时，按输出列的冲突文件，手动删 `<<<<<<< ======= >>>>>>>` 标记 → `git add <文件> && git rebase --continue`；搞不定就 `git rebase --abort` 回退重来
4. **secrets 不入库**：`.env`、token、密码绝不写进记忆文件
5. **新写入下个会话才生效**：你刚 capture 的，当前对话不会立即依赖它

## 出错处理

| 现象 | 修复 |
|---|---|
| `author 必须含 ascii 字母数字` | `<短名>` 用英文。`--author <短名>` 或 `.env` 的 `DEFAULT_AUTHOR=<短名>` |
| `team-memory 不是命令`（Windows） | 用 `python -m team_memory <命令>` |
| `sync` 报"本地有未提交变更" | 先 `git add -A && git commit`，或 `team-memory sync --force` |
| `sync` 报冲突 | 见上"工作约定"第 3 条 |
| `doctor` 报"解析失败" | frontmatter 坏了，用 `team-memory show <id>` 定位，手动修或重新 `capture` |
| web 打不开 | 端口被占：`team-memory web -p ~/team-memory --port 8001` |
