# 新人接入指南

> 我是团队新成员，团队已有共享记忆仓库（在 Gitea 上），我怎么加入并开始用？
> 本文档区分**两条路径**：① 接入已有仓库（多数人）② 首次创建仓库（仅管理员一次）。

---

## 前置：你需要什么

- 团队 Gitea 的账号 + 一个 **access token**（头像 → 设置 → 应用 → 生成令牌，勾选 `repository`）
- 本机装了 Python ≥ 3.11
- 你常用的 AI 工具（Claude Code / Codex / Hermes / Trae 任意组合）
- **你的英文短名**（用于记忆 id 的 author 段，如 `zhang3`、`vayne`；中文不行，因为要进 id）

---

## 路径一：接入已有团队仓库（推荐，多数人）

### 一键脚本（最省事）

**Mac / Linux**：
```bash
./scripts/onboard.sh <记忆仓库git地址> <你的英文短名>
# 或跨平台版:
python3 scripts/onboard.py <记忆仓库git地址> <你的英文短名>
```

**Windows**（用跨平台 Python 版）：
```powershell
python scripts/onboard.py <记忆仓库git地址> <你的英文短名>
# 例:
python scripts/onboard.py http://gitea.example.com/team/team-memory.git zhang3
```

脚本会自动：clone 记忆仓库 → 装 team-memory 工具 → 配 `.env` → 桥接到 4 工具 → 自检。
`onboard.py` 内部用 `python -m team_memory` 调命令，**不依赖 team-memory 是否进 PATH**，Windows 上最稳。

### 手动步骤（想自己掌控每一步）

```bash
# 1. clone 记忆仓库
git clone http://gitea.example.com/team/team-memory.git ~/team-memory
cd ~/team-memory

# 2. 装 team-memory 工具(从工具仓库)
pip install -e /path/to/agent-team-memory

# 3. 配 .env(从模板)
cp .env.example .env
# 编辑 .env: 填 GITEA_REMOTE_URL(已是仓库地址) + DEFAULT_AUTHOR=你的英文短名 + GITEA_TOKEN

# 4. 桥接到你本机的 AI 工具(装全局 skill)
team-memory load --global -p ~/team-memory

# 5. 自检
team-memory doctor -p ~/team-memory
```

完成后，你的 Claude Code / Codex / Hermes / Trae 都能读到团队记忆（通过各自的全局 skill）。

---

## 路径二：首次创建团队仓库（仅管理员，全团队只做一次）

```bash
# 1. 在 Gitea 建空私有仓库 team-memory(不要勾选 Initialize)
# 2. 本地 init
team-memory init -p ~/team-memory -n "你的团队名" -a <你的短名>
# 3. 配 .env 的 GITEA_REMOTE_URL 指向刚建的仓库, load --global, push
team-memory load --global -p ~/team-memory
cd ~/team-memory && git add -A && git commit -m "init" && team-memory sync
```

详见 [Gitea本地部署与迁移指南.md](Gitea本地部署与迁移指南.md) 和 [系统说明.md](系统说明.md)。

## 日常用法

| 想做什么 | 命令 |
|---|---|
| 沉淀一条记忆 | `team-memory capture "标题" -t decision --via claude --note "..."` |
| **对话中低门槛沉淀** | 直接对 AI 说"**提交下**" → auto-commit-memory 自动总结、确认后入库 |
| 看所有记忆 | `team-memory list` |
| 看某条详情 + 版本历史 | `team-memory show mem-20260625-zhang3-001` |
| 可视化（检索框 + 来源/版本） | `team-memory web` |
| 同步到 Gitea（拉队友最新 + 推自己的） | `team-memory sync` |
| 自检仓库健康 | `team-memory doctor` |
| 批量校验（CI 同款） | `team-memory validate` |

> 每次 `capture` 后记得 `git add/commit` 再 `sync`（capture 不自动 commit，人工把关）。

## 4 工具桥接说明

`load --global` 会在各工具装 `team-memory` skill + `auto-commit-memory` skill：

| 工具 | 位置 | 已验证 |
|---|---|---|
| Claude Code | `~/.claude/skills/team-memory/`、`~/.claude/skills/auto-commit-memory/` | ✅ 已实测触发 |
| Hermes | `~/.hermes/skills/team-memory/`（同构） | ✅ skill 体系同构 |
| Trae | `~/.trae/skills/` | ⚠ 需团队实测；若不识别，fallback 写 `~/.trae/rules/team-memory.md` |
| Codex | `~/.codex/skills/` + `~/.codex/AGENTS.md` 标记段 | ⚠ 需团队实测；触发式 skill 在 Codex 的支持待验证 |

未安装的工具会自动跳过。

## 常见问题

**Q: capture 报 "author 必须含 ascii 字母数字"**
A: 你的 author（或 `.env` 的 `DEFAULT_AUTHOR`）是中文/空。配一个英文短名：`echo 'DEFAULT_AUTHOR=zhang3' >> ~/team-memory/.env`，或 capture 时 `--author zhang3`。

**Q: 别人沉淀了新记忆，我怎么看到？**
A: `team-memory sync`（拉最新）→ `team-memory load --global`（刷新本机工具的 skill 清单）。或直接 `team-memory web` 浏览。

**Q: sync 报冲突怎么办？**
A: 按提示：手动编辑冲突文件删冲突标记 → `git add <文件> && git rebase --continue`；或 `git rebase --abort` 放弃重来。**不要**在冲突状态下继续 capture。
