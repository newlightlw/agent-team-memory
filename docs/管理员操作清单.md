# 管理员操作清单 — 团队记忆系统部署

> **给 AI 工具（Cursor/Claude Code 等）执行**：你是团队管理员。按本清单把 team-memory 部署到团队 Gitea。**全团队只做一次**。遇到 `<占位符>` 时询问用户填入真实值，不要自行编造。

---

## 前置条件

- Gitea 已运行（部署见 [`Gitea本地部署与迁移指南.md`](Gitea本地部署与迁移指南.md)），你有**管理员账号**
- 本机 Python ≥ 3.11
- 本机已 `git clone` 了 `agent-team-memory` 工具仓库（或拿到它的压缩包）

## 占位符（执行前确认，逐个问用户）

| 占位符 | 含义 | 示例 |
|---|---|---|
| `<GITEA>` | Gitea 根地址 | `http://gitea.example.com` 或 `http://localhost:3000` |
| `<ORG>` | 组织名 | `ai-team` |
| `<TEAM_NAME>` | 团队显示名 | `AI Native 团队` |
| `<ADMIN>` | 管理员英文短名（用于 id） | `vayne` |
| `<ADMIN_TOKEN>` | 管理员 Gitea token（scope: `repository` + `admin:org`） | `3db8...` |

## 步骤

### ① 建 Gitea 组织 + 两个仓库

```bash
# 建组织
curl -s -X POST <GITEA>/api/v1/orgs \
  -H "Authorization: token <ADMIN_TOKEN>" -H "Content-Type: application/json" \
  -d '{"username":"<ORG>","visibility":"private"}'

# 记忆仓库(数据, 私有, 空仓库 — 不要 auto_init)
curl -s -X POST <GITEA>/api/v1/orgs/<ORG>/repos \
  -H "Authorization: token <ADMIN_TOKEN>" -H "Content-Type: application/json" \
  -d '{"name":"team-memory","private":true,"description":"团队共享记忆仓库","auto_init":false}'

# 工具仓库(代码)
curl -s -X POST <GITEA>/api/v1/orgs/<ORG>/repos \
  -H "Authorization: token <ADMIN_TOKEN>" -H "Content-Type: application/json" \
  -d '{"name":"agent-team-memory","private":true,"description":"team-memory CLI 工具"}'
```
**验证**：`curl -s <GITEA>/api/v1/repos/<ORG>/team-memory` 返回 JSON（HTTP 200）。

### ② 推工具仓库到 Gitea

```bash
cd /path/to/agent-team-memory   # 本地工具仓库
git remote add origin <GITEA>/<ORG>/agent-team-memory.git
git push -u origin main
```
**验证**：浏览器打开 `<GITEA>/<ORG>/agent-team-memory`，能看到代码和 8 个 commit。

### ③ 初始化记忆仓库 + 推首条记忆

```bash
pip install -e .                                   # 装工具
team-memory init -p ~/team-memory -n "<TEAM_NAME>" -a <ADMIN>
# 配 .env 指向 Gitea
#   编辑 ~/team-memory/.env: GITEA_REMOTE_URL=<GITEA>/<ORG>/team-memory.git
team-memory capture "团队记忆系统启动" -t project --via claude -a <ADMIN> \
  -p ~/team-memory --note "<TEAM_NAME> 团队共享记忆仓库启动, 四工具(Claude/Codex/Trae/Hermes)共用。"
team-memory load --global -p ~/team-memory
cd ~/team-memory
git add -A && git commit -m "chore: 初始化团队记忆仓库"
team-memory sync                                   # 推到 Gitea
```
**验证**：`curl -s <GITEA>/api/v1/repos/<ORG>/team-memory/contents` 返回根目录文件列表。

### ④ 添加组员到组织 + 指导生成 token

- 把团队成员加入 `<ORG>` 组织（Gitea 网页：组织 → 设置 → 团队 → 添加成员）
- 通知每位组员**自己生成 token**：头像 → 设置 → 应用 → 生成新令牌 → 勾选 `repository`
- 给每位组员分配一个**唯一的英文短名**（用于记忆 id 的 author 段，如 `zhang3`、`lisi`，不能中文、不能撞）

### ⑤ 启用 Gitea Actions CI（推荐）

- 配置 Gitea Actions runner（见 Gitea 官方文档：Site Administration → Actions → Runners）
- `.gitea/workflows/validate-memory.yml` 已在工具仓库，记忆仓库需复制一份并改 `pip install` 地址指向 `<GITEA>/<ORG>/agent-team-memory.git`

**验证**：故意提一个 frontmatter 残缺的 PR → workflow `校验记忆文件` 失败、阻止合并。

### ⑥ 把"组员操作清单"发给全员

把 [`组员操作清单.md`](组员操作清单.md) 发到团队群。附上：
- `<GITEA>` 地址、`<ORG>` 组织名
- 各人的英文短名
- 让组员把该清单交给他们的 AI 工具执行

### ⑦ 验收

```bash
# 看多人是否已接入(提交记录)
curl -s <GITEA>/api/v1/repos/<ORG>/team-memory/commits | python -c "import sys,json;d=json.load(sys.stdin);print(len(d),'个提交')"
```
- 抽查 1-2 位组员本机：`team-memory doctor -p ~/team-memory` 应健康
- `team-memory list` 应能看到不同 `source`/`author` 的记忆

## 出错处理

| 现象 | 原因 / 修复 |
|---|---|
| API 401/403 | token scope 不足，需 `repository` + `admin:org` |
| push 403 | 组员未被加入 `<ORG>` 组织，或 token 无写权限 |
| `init` 报 author 非 ascii | `<ADMIN>` 用英文短名 |
| CI 不触发 | runner 未注册；workflow 文件未在记忆仓库的 `.gitea/workflows/` |
| 组员 id 撞号 | 不会发生（id 含 author）；若仍撞，检查两人是否用了相同短名 |
